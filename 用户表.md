# 用户表

## TL;DR
```sql
-- 用户认证表, 主要处理用户注册, 登录, 重置密码的逻辑
CREATE TABLE users (
    uid             BIGINT PRIMARY KEY,
    username        VARCHAR(64) NOT NULL,
    phone           VARCHAR(20),
    email           VARCHAR(255),
    password_hash   VARCHAR(255) NOT NULL,
    status          SMALLINT NOT NULL DEFAULT 1,         
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMPTZ                            
);

```

## 为什么没有用户昵称, 性别等常见信息?
我倾向于将这类信息放到 user_public_profiles 表中
这样做的优点是: 
1. 保证每张表的职责单一
2. profile 更新时, 只会锁住 profile 表并产生一份冗余的 profile 记录

## 软删除
大多数表中, 我都会使用软删除来保留数据痕迹, 这样当错误删除发生时, 可以快速恢复

需要注意唯一索引可能会和 软删除 冲突, 需要使用唯一部分索引
```
-- 唯一约束改为部分索引，仅对未删除数据生效
CREATE UNIQUE INDEX idx_users_username_active ON users (username) WHERE deleted_at IS NULL;
CREATE UNIQUE INDEX idx_users_phone_active    ON users (phone)    WHERE deleted_at IS NULL;
CREATE UNIQUE INDEX idx_users_email_active    ON users (email)    WHERE deleted_at IS NULL;
```
